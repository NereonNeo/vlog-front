# План реализации

Дата актуализации: 2026-02-18

## 0) Подготовка и контент

- Уточнить финальный набор страниц: `/`, `/projects`, `/projects/[slug]`, `/posts`, `/posts/[slug]` (опционально `/about`, `/tags/[tag]`).
- Подготовить первичные данные для `projects` и `posts` (минимум по 3–4 записи).
- Уточнить тексты для Hero и CTA на главной.
- Согласовать список технологий/тегов, статусов проектов, ролей.

## 1) Основа приложения и инфраструктура

- Проверить версию Next.js и структуру приложения (App Router/Pages Router).
- Настроить базовые метаданные и SEO-скелет:
  - `sitemap.xml`, `robots.txt`, `canonical`.
  - OpenGraph шаблоны для постов и проектов.
- Подготовить структуру директорий для данных, компонентов и страниц.
- Добавить базовые настройки линтинга/форматирования (если есть пробелы).

Артефакты:

- Черновые маршруты страниц и пустые шаблоны.
- Базовый SEO-скелет и метаданные.

## 2) Дизайн-система и глобальные стили

- Определить палитру (background/surface/border/accent/text).
- Задать типографику (шрифты для заголовков/текста/кода).
- Настроить глобальные CSS переменные и базовые состояния (hover/focus).
- Подготовить стеклянные поверхности, тени, радиусы, микро-градиенты и шум.

Артефакты:

- Глобальные стили и токены дизайна.
- Базовые утилитарные классы/миксины (если используются).

## 3) Базовые UI-компоненты

- Реализовать компоненты:
  - `Card`, `Tag`, `Badge`, `ReadingTimeWidget`, `Breadcrumbs`.
  - `FilterBar`, `EmptyState`, `MDXContent`.
- Добавить состояния:
  - Loading (skeleton), Empty, Error.
  - Полноценные focus-стили для клавиатуры.

Артефакты:

- Набор переиспользуемых компонентов в `src/components`.

## 4) Контент-модель и данные

- Описать типы `Project` и `Post` (TS types).
- Организовать хранение:
  - Локальные JSON/TS-данные или файловый контент (MD/MDX).
- Реализовать расчет `readingTimeMinutes`.
- Добавить связку проектов и постов (по тегам или explicit links).

Артефакты:

- Типы и функции выборок данных.
- Моки/реальные данные для страницы.

## 5) Реализация страниц

### 5.1 Главная `/`

- Секции: Hero, Skills, Featured Projects, Latest Posts, Contacts.
- Полная кликабельность карточек.

### 5.2 Список проектов `/projects`

- Сетка карточек + `FilterBar`.
- Фильтры: статус, технологии, поиск.
- Сортировки: newest, most impactful, by status.

### 5.3 Детальная проекта `/projects/[slug]`

- Hero проекта + факты.
- Блоки: О проекте, Вклад, Стек, Архитектура/решения, Статус и планы.
- Related posts.

### 5.4 Список постов `/posts`

- Поиск + фильтры по тегам + сортировка.
- Пагинация или аккуратный infinite scroll (подтвердить выбор).

### 5.5 Детальная поста `/posts/[slug]`

- Заголовок + мета (дата, reading time, теги).
- Контент MDX/Markdown + Next/Prev + Related.

Артефакты:

- Полный набор страниц с маршрутизацией.

## 6) Поведение, интеракции и доступность

- Полная поддержка клавиатурной навигации.
- Hover/focus состояния для интерактивных элементов.
- Проверка aria-атрибутов для фильтров и навигации.
- Проверка CLS (стабильные размеры медиа и карточек).

Артефакты:

- А11y-проверки и базовые исправления.

## 7) Производительность и SEO

- Оптимизация изображений, lazy-loading.
- Проверка Lighthouse (perf/seo/a11y >= 90).
- Финальная проверка метаданных, OG, sitemap.

Артефакты:

- Отчет по Lighthouse и список точечных улучшений.

## 8) Полировка и финальная приемка

- Тестирование на мобиле/десктопе.
- Визуальная проверка соответствия “dark fantasy” концепции.
- Финальные правки и чистка TODO.

Артефакты:

- Готовый к публикации проект.

## Риски и решения

- **Недостаток контента**: заранее подготовить минимум по 3–4 проекта/поста.
- **Сложность фильтров**: на первом этапе упростить (теги + поиск), расширить позже.
- **Перегруз дизайна**: держать акцент на строгости, избегать лишних эффектов.

## Критерии готовности (Definition of Done)

- Все страницы доступны и соответствуют маршрутам из требований.
- Есть заполненный контент и корректные данные.
- Lighthouse показатели >= 90.
- Клавиатурная навигация и фокус-стили корректны.
- SEO-метаданные и OG настроены.
