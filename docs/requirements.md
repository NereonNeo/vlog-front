## 1) Цели и принципы

### 1.1 Цели

- Показать тебя как инженера: **кто ты**, **что сделал**, **как думаешь**, **чем владеешь**.
- Сделать статьи “вторым резюме”: архитектура, разборы решений, заметки, туториалы.
- Дать быстрый путь рекрутеру/заказчику:
  - 10 секунд: “кто он и чем полезен”
  - 30 секунд: “какие проекты и стек”
  - 2–5 минут: “деталь проекта/статьи”

### 1.2 Тон и визуальная идея (dark fantasy, но строгая)

- Визуальный образ: **обсидиан + дымчатое стекло**, не как декоративный шум.
- Иерархия: большие заголовки, чистая сетка, много воздуха.
- Детали: лёгкие градиенты, микрошум, подсветка краёв карточек, аккуратные hover-состояния.

## 2) Информационная архитектура (страницы и маршруты)

### 2.1 Основные страницы

1. **Главная** — `/`
2. **Проекты (список)** — `/projects`
3. **Проект (детально)** — `/projects/[slug]`
4. **Посты (список + фильтры)** — `/posts`
5. **Пост (детально)** — `/posts/[slug]`

### 2.2 Дополнительно (рекомендуется)

- `/about` (если хочешь отделить историю/био от главной)
- `/tags/[tag]` (SEO-дружелюбная страница для тегов, но можно и без неё — фильтрация на `/posts`)

## 3) Общие UI-компоненты и поведение

### 3.1 Layout

**Header (липкий)**

- Лого/ник
- Навигация: Home / Projects / Posts
- Переключатель темы (опционально: “Night / Abyss”)
- Кнопка “Контакты” или “Hire me” (модалка или секция в футере)

**Footer**

- Контакты: GitHub / Telegram / Email / LinkedIn
- Copyright
- “Собрано в …” (не обязательно)

### 3.2 Основные компоненты

- `Card` (проект, пост)
- `Tag` (чип)
- `Badge` для статуса проекта (Active / Maintained / Archived)
- `ReadingTimeWidget` (минуты чтения)
- `Breadcrumbs` на детальных страницах
- `FilterBar` (поиск + теги + сортировка)
- `EmptyState` (когда фильтр ничего не нашёл)
- `MDXContent` (рендер поста + стилизация typograpy)

### 3.3 Состояния

- Loading (skeleton)
- Empty
- Error (например, пост не найден)
- Hover/Focus (обязательно keyboard-friendly)

## 4) Документация по сценариям (что именно должно быть)

### Сценарий A — Главная страница (основная информация обо мне)

**Цель:** за 10–20 секунд объяснить кто ты и почему ты сильный.

**Секции:**

1. **Hero**
   - Имя/ник + роль (например: “Frontend Engineer / Full-stack leaning”)
   - 1–2 предложения “что я делаю” (без воды)
   - CTA: “Смотреть проекты” + “Читать статьи”

2. **Ключевые компетенции**
   - 6–10 “плашек”: TypeScript, React, Next.js, Node, DB, CI/CD, etc.
   - (В стиле: “гримуар навыков” но строгий)

3. **Featured projects (3–4 карточки)**
   - Название, 1 строка описания, стек-иконки/теги, статус

4. **Последние посты (3–5)**
   - Название, дата, теги, reading time

5. **Контакты / Social**
   - Коротко и аккуратно (иконки + ссылки)

**UX правила**

- На главной минимум текста-стены. Лёгкий скролл = чувство глубины.
- Любая карточка кликабельна целиком.

---

### Сценарий B — Проекты, список (участие + стек + статус)

Страница `/projects`

**Функции**

- Сетка карточек (2–3 колонки на десктопе)
- Фильтры:
  - по статусу (Active / Maintained / Archived)
  - по технологиям (React, Next, Node…)
  - поиск по названию

- Сортировка:
  - Newest
  - Most impactful (ручной рейтинг)
  - By status

**Карточка проекта (минимальный набор)**

- Название
- Короткое описание (1–2 строки)
- Стек: 3–8 тегов (или “Top 3 + ещё N”)
- Статус (бейдж)
- Роли/участие (например: “Frontend / Architecture / DevOps”) — компактно
- Метаданные: год/период

---

### Сценарий C — Каждый проект отдельная страница

`/projects/[slug]`

**Структура**

1. **Hero проекта**
   - Название
   - Статус (Active/Archived)
   - Период, роль, команда (если уместно)
   - Ссылки: demo / repo / case-study (если есть)

2. **О проекте**
   - 1–2 абзаца: что это и зачем

3. **Мой вклад**
   - буллеты: что именно сделал (конкретика: “внедрил кеш-ключи”, “переписал авторизацию”, “оптимизировал картинки в webp”)

4. **Стек**
   - чипы технологий + группировка:
     - Frontend
     - Backend
     - Infra/Tools

5. **Архитектура / Решения (опционально, но очень круто)**
   - схемы/картинки (если будут)
   - ключевые trade-offs

6. **Статус и планы**
   - Active: roadmap
   - Archived: почему, чему научился

7. **Связанные посты**
   - подборка статей с тегом проекта

**UX детали**

- На правой стороне (или сверху на мобиле): “Project facts” как небольшая стеклянная панель:
  - status, role, stack count, links

---

### Сценарий D — Страница с постами и фильтрацией по тегам

`/posts`

**Функции**

- Search input (по заголовку + краткому описанию)
- Фильтр по тегам (multi-select)
- Сортировка:
  - newest
  - oldest
  - reading time (короткие/длинные)

- Пагинация или infinite scroll (выбирай; для строгого стиля — пагинация аккуратнее)

**Карточка поста**

- Заголовок
- Дата
- “Reading time”
- Теги (2–5)
- Короткое описание/аннотация (1–2 строки)

**Empty state**

- “В этом гримуаре нет записей по этим рунам” (можно так, но без клоунады)

---

### Сценарий E — Детальная страница поста

`/posts/[slug]`

**Структура**

1. Заголовок + мета:
   - дата
   - reading time (виджет)
   - теги (виджет)

2. Контент MDX/Markdown
3. “Следующий/Предыдущий пост”
4. “Похожие по тегам”

---

### Сценарий F — Виджеты “Время для чтения” и “Теги”

**Reading time**

- Логика:
  - считаем количество слов
  - делим на скорость чтения (например, 200–240 wpm)
  - округляем вверх до минут

- Отображение:
  - “~ 6 min read”
  - Иконка песочных часов/свечи (строго, тонко)

**Tags**

- Чипы с hover glow
- Клик по тегу:
  - ведёт на `/posts?tags=...` или `/tags/[tag]`

## 5) Дизайн-система (строго + dark fantasy)

### 5.1 Цвета (концепт)

- Background: почти чёрный с глубоким оттенком (не чистый #000)
- Surface (карточки): тёмный “стеклянный” слой с прозрачностью
- Border: тонкая, чуть светлее поверхности
- Accent: бронза/старое золото (один акцент, не радуга)
- Text:
  - primary: почти белый
  - secondary: серо-дымчатый

### 5.2 Форма

- Радиусы: 14–20px (карточки), 10–14px (инпуты/чипы)
- Тени: мягкие, небольшие, без грязи
- Прозрачность: 6–14% (только на поверхностях)

### 5.3 Типографика

- Заголовки: строгий шрифт (геометрический/антиква-намёк, но читаемый)
- Текст: нейтральный (читаемый)
- Код-блоки: моноширинный, аккуратные границы

### 5.4 Эффекты (минимально)

- Hover: лёгкая подсветка бордера + микросдвиг 1–2px
- Background texture: еле заметный шум/градиент

## 6) Данные и контент-модель (что хранить)

### 6.1 Project (сущность)

- `slug` (string)
- `title`
- `summary`
- `description` (long)
- `status` = `active | maintained | archived`
- `period` (start/end)
- `roles` (array)
- `stack` (array of tags/tech)
- `links` (demo/repo/etc)
- `featured` (boolean)
- `highlights` (array) — “мой вклад”
- `relatedPostSlugs` (array) (или вычислять по тегам)

### 6.2 Post (сущность)

- `slug`
- `title`
- `description` (аннотация)
- `date`
- `tags` (array)
- `content` (md/mdx)
- `readingTimeMinutes` (computed)
- `cover` (optional)

## 7) Контент-стратегия (чтобы сайт выглядел живым)

- 3 типа постов:
  1. “Case study” по проекту (архитектура/решения)
  2. “How-to” (практика)
  3. “Notes” (короткие мысли/наблюдения)

- На главной показывать “Featured” проекты и “Latest” посты.

---

## 8) Нефункциональные требования (чтобы не было стыда)

- Lighthouse: performance/seo/accessibility — не ниже 90 (идеально)
- Адаптивность: мобила обязательна
- Keyboard navigation: фокус-стили в теме
- SEO:
  - OpenGraph для постов и проектов
  - canonical
  - sitemap.xml

- Контент должен рендериться быстро и без “скачков” (CLS)

- Слева/сверху FilterBar

- Вверху: title + status + links
